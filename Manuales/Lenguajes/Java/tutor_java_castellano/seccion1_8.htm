Sección 1.8: ¿Porqué Java?, ¿Porqué no C++?

En el pasado (otoño de 1996), Estas notas utilizaron el Pascal como lenguaje de programación. A pesar de que el Pascal es un lenguaje muy agradable, la ciencia de los ordenadores, empujada por la moda, lo abandonó definitivamente para usar un lenguaje mas "serio". Muchos programadores profesionales estaban usando el lenguaje C y C++ y los estudiantes estaban ansiosos por aprender esos lenguajes. Los departamentos de ciencias de la computación de una gran cantidad de universidades realizaron el cambio. Y también después de muchas discusiones y alguna pena, el Departamento de Matemáticas y ciencias de la computación de Hobart and William Smith Colleges decidieron cambiar a C++ como lenguaje de instrucciones en estas clases de introducción a la programación.

La decisión se tomo con mucho pesar, porque precisamente el C++ no es el mejor lenguaje de programación para programadores novatos. Es un lenguaje grande y complejo con muchas características que son la delicia de los programadores con experiencia. Pero son precisamente estas características las que dificultan el aprendizaje a los principiantes.. De hecho, durante el curso, solo se les puede mostrar una pequeña parte del conjunto de instrucciones de C++.Aun así, todas esas características siguen ahí y no es posible evitar completamente todas las complicaciones que causan.

Ahora Java. Java es un lenguaje de programación bastante nuevo, pero ya parece probable que se vaya a convertir en un lenguaje muy importante--quizás aun mas importante que el C++ a pesar de su larga carrera.Java comparte gran numero de las características básicas de C++. En cambio, omite aquellas otras que utilizan los programadores avanzados, pero   que tienden a causar confusión y problemas , especialmente entre los novatos. Debido a que comparte muchas cosas, debe ser relativamente sencillo pasar de Java a C++. De hecho, C++ se emplea como lenguaje de instrucciones  en CS125 , el curso siguiente a este. En el Capítulo 9, se analizan alguna de las características avanzadas del C++

Ahora, no se le ocurra pensar que Java es un lenguaje inferior, únicamente porque es mas simple que el C++. De hecho, Java tiene algunas características que logran mejorar muchos aspectos del C++.Es mas potente, naturalmente, es capaz de ser descargado por la red, puede funcionar  en gran numero de plataformas distintas, y puede trabajar conjuntamente con el navegador. Estas habilidades hacen que provoque tanto apasionamiento.

Además, Java fue construido con soporte para "hebras" (threads) como se menciono en la Sección 2 . Este soporte hace fácil el escribir programas que deban hacer varias cosas a la vez (tal como navegar, poder descargar un fichero, redibujar la pantalla y seguir un nuevo link al mismo tiempo--mientras funciona un pequeño applet de Java, es una buena medida).

Otras de las características de Java que muchas personas ven como ventaja, en la recogida de desperdicios. Voy a intentar explicarlo ahora, pero tiene que ver con la forma en la que el programa utiliza la memoria del ordenador. En C++, los programadores deben de trabajar bastante con el manejo de la memoria, esta tarea es difícil y con tendencia a provocar errores. Java utiliza un sistema de recogida de desperdicios, que automatiza todo el manejo de la memoria (Los "desperdicios" es la memoria que ha utilizado el programa pero que ya no la necesita). Naturalmente, el programador de C++ puede afirmar que el programador de Java pierde gran parte del control sobre la maquina. Es cierto,  tanto por la "recogida de desperdicios" como por otras características de Java. Todo esto hace de Java un lenguaje mas fácil y mas seguro, pero también es el motivo por el que es difícil que llegue a reemplazar el C++ totalmente.

En conjunto, Java es un lenguaje bien diseñado, agradable de usar y aun así ofrece mucha potencia.Llegue a ser tan importante como creo o no,  parece ser un buen lugar para que empiecen los programadores novatos.
