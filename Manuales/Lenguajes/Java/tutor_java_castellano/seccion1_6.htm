Sección 1.6: La moderna interface de usuario

Cuando los ordenadores empezaron a aparecer, la mayoria de las personasl, e incluso muchos programadores, no podían estar cerca de ellos. Los ordenadores, se situaban en habitaciones cerradas, con asistentes vestidos de blanco que  tomaban sus programas y sus datos, para alimentar al ordenador, devolviendo la respuesta de la maquina al cabo de un tiempo. Cuando en 1960 se invento el "tiempo compartido" --el sistema por el  que el ordenador puede ir  pasando la atención de uno al otro usuario rápidamente--, apareció la posibilidad que mucha gente interactuara con el ordenador al mismo tiempo. En los sistemas de tiempo compartido, el usuario esta en un terminal tecleando comandos, mientras que el ordenador le va entregando las respuestas. Los antiguos asistentes del ordenador, trabajaban de la misma manera, lo único que ahora se podía atender mas de una petición al tiempo. La forma de interacción que estamos describiendo, se llama interface de línea de comando (Command-line interface).

Hoy en día, la mayoría de personas trabajan con el ordenador de una manera muy distinta. Usan lo que se llama una interface gráfica o GUI de Graphical User Interface. Los ordenadores dibujan en la pantalla componentes de interface. Estos componentes incluyen cosas como ventanas, barras de desplazamiento, menús e iconos. Normalmente, es posible usar el ratón para interactuar con estos componentes. Asumimos que si usted esta leyendo estas notas en un ordenador, esta sin duda familiarizado con las bases de la interface gráfica de usuario.

Una gran cantidad de estos componentes gráficos, se han convertido en estándares.Esto es, tienen una apariencia y un comportamiento similar, en ordenadores y plataformas diferentes, incluidas Macintosh, Windows 3.1, Windows 95 y algunos sistemas UNIX. Los programas Java, que se supone, pueden funcionar en distintas plataformas sin modificar el código, pueden usar los componentes estándares GUI.: Pueden variar de apariencia de una plataforma a otra, pero su funcionalidad debe ser idéntica en cualquier ordenador en el que funcione el programa.

Puede ver como los mensajes se van presentando en el área de texto, y experimentar con otros componentes.Lo que ocurre, es que cuando realiza ciertas acciones, como por ejemplo, pulsar un botón, se genera un evento. Un mensajes es trasmitido hacia el applet comunicando el evento aparecido, y el applet responde tal y como se le programado, en este caso displayando mensajes en el área de texto.

El emplear aquí el termino "mensaje", es deliberado.Tal y como se dijo en la sección precedente, los objetos se comunican por mensajes. De hecho, Java implementa los componentes GUI como objetos. Java incluye gran cantidad de clases GUI predefinidas, y que pueden ser usadas en los programas Java. Algunas de estas clases, son subclases de otras. A continuación, ofrecemos el diagrama que presenta las relación de algunas de estas clases.

No se preocupe ahora por los detalles. Pero note que todas las clases GUI son directa o indirectamente subclases de una llamada Componente: Dos de las subclases directas de Componente, tienen a la vez subclases. Las clases TexArea y TextField, que ciertamente tienen cosas en común, han sido agrupadas en una subclase TextComponent. La clase denominada Container, se refiere a un componente que puede contener otros componentes. La clase Applet es, indirectamente, una subclase de Container puesto que un applet puede contener tanto botones, como campos de texto.

Hasta aquí esta breve presentación, puede que ya conozca como se puede hacer un uso efectivo en la  programación, del diseño orientado a objetos de GUI.. Quizás sea GUI, al ser un objeto visible, lo que mas haya contribuido a popularizar la OOP.
