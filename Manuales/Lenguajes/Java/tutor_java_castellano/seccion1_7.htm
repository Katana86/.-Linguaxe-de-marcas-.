Sección 1.7: Internet y el World-Wide Web

Los ordenadores, se pueden conectar en redes. Un ordenador conectado a una red, puede intercambiar ficheros y datos o enviar y recibir mensajes con cualquier otro ordenador que también este conectado a la red. Los ordenadores conectados en red, pueden trabajar conjuntamente para realizar grandes cálculos.

Cada día, millones de ordenadores de todas partes del mundo, son conectados a una red gigante que se llama Internet. Cada día se conectan mas ordenadores a Internet. Y de hecho los ordenadores, se pueden unir temporalmente a Internet usando un módem, estableciendo la comunicación por medio de las líneas telefónicas.

Existen protocolos de comunicación creados para la comunicación en Internet. Un protocolo es, simplemente, un conjunto de especificaciones detalladas sobre el procedimiento a seguir para comunicarse. Para que dos ordenadores se puedan comunicar entre ello, deben estar utilizando el mismo protocolo. El protocolo mas básico es el Internet Protocol (IP), que especifica como son transmitidos físicamente los datos de un  ordenador a otro. y el Transmisión Control Protocol (TCP), que asegura que los datos que se han emitido usando IP, serán recibidos por completo y sin ningún error.Estos dos protocolos, que normalmente se les menciona conjuntamente como TCP/IP, establecen la base para las comunicaciones. También  otros protocolos usan TCP/IP para manejar otras informaciones especificas como la transmisión de ficheros y el correo electrónico.

Las comunicaciones en Internet, son en forma de paquetes. Un paquete consiste en una parte de datos unida a otra parte de información sobre la dirección, que indica a que punto de la red Internet se supone deben ir esos datos. Podemos pensar en los paquetes como en un sobre con la dirección fuera, y el mensaje dentro. Los paquetes también incluyen la dirección de devolución, esto es la dirección del emisor. Un paquete, solo puede almacenar una cantidad limitada de información; los mensajes largos, deben ser divididos en varios paquetes, que serán transmitidos individualmente, y en destino, se volverán a montar.

Muchos ordenadores tienen una dirección IP en Internet, esto es, un numero que es único y que identifica inequívocamente el ordenador en la red. La dirección IP se utiliza para direccionar los paquetes, luego en Internet, un ordenador solo puede enviar datos a otro sí previamente conoce su dirección IP. Dado que las personas prefieren utilizar los nombres mas que los números,  muchos ordenadores pueden ser identificados por un nombre , llamado nombre de dominio. Por ejemplo, el ordenador principal de Hobart and William Smith Colleges tiene como nombre de dominio hws3.hws.edu. (Los nombres de dominio, se utilizan para nuestra comodidad; el ordenador va a necesitar conocer la dirección IP antes de iniciar la comunicación. Por eso existen ordenadores en Internet cuyo trabajo consiste en convertir los nombres de dominio en direcciones IP.Cuando usa un nombre de dominio, su ordenador envía un mensaje a un servidor de nombres para buscar la dirección IP que le corresponde).

Internet proporciona una serie de servicios a los ordenadores conectados ( y como es natural a los usuarios de esos ordenadores). Estos servicios usan TCP/IP para enviar distintos tipos de datos sobre la red.Alguno de los mas populares servicios son Telnet, Correo electrónico (E-MAIL), FTP, gopher y el World-Wide Web.

Telnet permite al usuario de un ordenador conectarse como usuario en otro ordenador. (Naturalmente, esta persona necesita conocer el código de usuario y la palabra de paso correspondiente a la cuenta del otro ordenador). Esencialmente, el primer ordenador actúa como terminal del segundo. Telnet, lo usa normalmente las personas que necesitan acceder al ordenador de su casa o de su oficina desde otras ubicaciones; en este caso lo pueden hacer con cualquier ordenador conectado a Internet , desde cualquier lugar del mundo.

Correo electrónico o E-MAIL facilita una comunicación sobre Internet de persona a persona. Un mensaje e-mail se emite por un usuario concreto de un ordenador, para un usuario concreto de otro ordenador. Cada persona se identifica por una dirección e-mail única, que consiste en el nombre de dominio del ordenador en donde este usuario tiene su cuenta de correo, juntamente con el nombre del usuario. La dirección e-mail tiene la forma "nombreUsuario@nombre.Dominio". Por ejemplo, mi dirección de correo electrónico es mgyea@arrakis.es. El e-mail se transmite desde un ordenador a otro usando un protocolo llamado SMTP (Simple Mail Transfer Protocol). El correo electrónico es el servicio de Internet mas usado y mas importante.

FTP (File Transport Protocol) esta diseñado para copiar ficheros entre dos ordenadores. Como en el caso de Telnet, el FTP requiere que el usuario conozca el nombre y el password para conectar con el otro ordenador. De todas maneras, muchos ordenadores disponen de una cuenta especial que permite acceder por medio de FTP con el nombre de usuario "anonymous" y cualquier password. Este sistema , que se llama FTP anónimo, se utiliza para publicar ficheros en Internet dejándolos al alcance de todo el mundo.

Gopher se desarrolló como una simplificación del FTP, con una interface de usuario mucho mas sencilla. En el ordenador que quiere publicar la información utilizando este protocolo, funciona un programa llamado "servidor Gopher". Cualquier persona, se puede conectar por medio de Internet, usando un programa llamado "cliente Gopher". Cuando el cliente conecta con el servidor,   el servidor le devuelve una lista de entradas  que el cliente presenta al usuario. Cada entrada representa un fichero, o una lista de entradas de este o de otro ordenador. Cuando el usuario selecciona un fichero,  el fichero se descarga desde el servidor y puede ser guardado y/o presentado en la pantalla del usuario. Cuando el usuario selecciona una entrada que representa otra lista, el cliente recibe la nueva lista y la presenta, pudiendo el usuario realizar otra selección. La parte excitante, es que esta nueva lista, puede provenir de un servidor Gopher en un ordenador diferente, en cualquier parte de Internet. El usuario Gopher, puede ir saltando de máquina a máquina, recuperando información de cualquier parte del mundo, sin tenerse que preocupar de direcciones IP, nombres de dominio, nombres de usuario, passwords o los misteriosos comandos que se usaban en los antiguos protocolos como FTP y Telnet.

El World-Wide Web (WWW) es algo semejante al Gopher llevada al extremo.Aunque el Gopher hacia que  para el usuario  Internet fuera fácil e incluso a veces divertida , solo permitía tratar ficheros y listas. El WWW esta basado en paginas que contienen distintos tipos de  información y también links (enlaces) a otras paginas. El montaje, es mucho mas sencillo que en Gopher. Para publicar una información en WWW, el ordenador ejecuta un programa generalmente llamado Web server. El usuario accede al Web usando un programa cliente, normalmente llamado Web browser o navegador.El navegador contacta con el servidor para recuperar una pagina de información. Si el usuario selecciona un enlace (link) a otra pagina, el navegador sigue el enlace para recuperar y presentar la nueva información. La idea es simple, sin embargo el resultado es complejo: El usuario tiene acceso a una descomunal cantidad de información entrelazada, por la que puede ir navegando sin necesidad de conocer profundamente los ordenadores.El Web es la parte mas interesante de Internet, y es la que la esta conduciendo al fenomenal ritmo de crecimiento que experimenta. Si todas las premisas se cumplen, la Web se convertirá en una parte universalmente fundamental de la vida de cada día.

Debe tener en cuenta que, aunque el FTP y el Gopher no han desaparecido todavía, han sido absorbidos dentro de World-Wide Web. El protocolo usado por los servidores Web se llama HTTP (HyperText Transfer Protocol), y los navegadores se comunican con los servidores empleando ese protocolo. Sin embargo, un navegador entiende varios protocolos, incluidos el FTP y el Gopher. Esto significa que un navegador puede comunicarse con servidores de FTP anónimos, y servidores Gopher con la misma facilidad que con servidores Web. La idea es que solo se necesite un programa con una interface para acceder a toda la información disponible en  Internet. Cuando esta usando un navegador, no se entera si usa FTP, Gopher o HTTP. Así debe ser: Usted quiere una información, el navegador ha de saber como conseguirla.

Justamente ahora, debe de estar pensando, ¿que tiene que ver todo esto con Java?.   De hecho, Java esta íntimamente asociada con Internet y con el World-Wide Web. Tal y como vio en la sección anterior, existen unos programas especiales Java, llamados applets, que pueden ser transmitidos por Internet y presentados en las paginas Web. Un servidor Web transmite los applets de Java igual que puede transmitir cualquier otro tipo de información. Un navegador que entienda Java-- esto es que incluya un interprete para la maquina virtual Java-- puede ejecutar correctamente el applet en medio de la pagina Web. Puesto que los applets son programas, permiten crear interacciones complejas con el usuario.  Con Java las paginas Web son mucho mas que una representación de información totalmente pasiva pudiendo llegar a convertirse en cualquier cosa que el programador pueda imaginar e implementar.
