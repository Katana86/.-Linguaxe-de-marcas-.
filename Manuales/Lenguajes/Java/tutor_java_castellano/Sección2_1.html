Sección 2.1:La aplicación Java básica.

Un programa es una secuencia de instrucciones que ejecuta un ordenador para realizar alguna tarea. Una idea bastante sencilla, pero para que un ordenador pueda hacerlo, deben estar escritas de tal forma que él   pueda usarlas. La forma de conseguirlo, es que el programa este escrito en código máquina. Los lenguajes de programación difieren del lenguaje humano, porque no son en absoluto ambiguos, y son muy estrictos al definir que cosas se puede hacer en un programa y cuales no. Las reglas que determinan que es lo que esta permitido escribir, se llama sintaxis del lenguaje. Las reglas de sintaxis, especifican el vocabulario básico del lenguaje y como se puede escribir el programa usando cosas como lo bucles los desvíos y las subrutinas. Cuando el programa esta correcto sintácticamente, se puede compilar o interpretar; si el programa tiene errores de sintaxis, se rechaza.(Se producen unos mensajes de error, que muchas veces ayudan a solucionar los errores).

Asi que para tener éxito programando, tiene que desarrollar un detallado conocimiento de la sintaxis del lenguaje de programación que este utilizando. De cualquier forma, la sintaxis, es solo una parte de la historia. No basta escribir un programa y conseguir que se ejecute. Usted querrá que el programa funcione y además, produzca un resultado correcto. Esto es, el programa ha de tener sentido. La semántica establece el sentido del programa. Una de las cosas que usted debe querer, es un programa semánticamente correcto.

Cuando en estas notas, se define una nueva característica del lenguaje, se explicara tanto la sintaxis como la semántica de esa característica. Deberá memorizar la sintaxis, esa es la parte fácil. Luego deberá intentar entender la semántica, por medio de los ejemplos que se le facilitan, asegurándose de comprender perfectamente como funciona e incluso haciendo algún pequeño programa para comprobar que lo ha entendido. Tenga en cuenta que conseguir familiarizarse con las características individuales del lenguaje no le hará ser un programador. Tiene que aprender como construir programas complejos para resolver problemas particulares. Por lo tanto, va a necesitar conocimientos y experiencia. En estas notas, solo encontrara el inicio del aprendizaje del desarrollo de software.

Empezaremos nuestra exploración de Java, con el problema que a menudo es el mas habitual para los principiantes: Escribir un programa que presente en pantalla el mensaje "Hola Mundo". Esto puede parecer un problema trivial, pero conseguir que el ordenador lo haga, es realmente un paso importante en nuestro aprendizaje de un nuevo lenguaje de programación (especialmente si este es su primer lenguaje de programación).Será la forma de que entienda los procesos básicos para:

  1. Introducir el texto del programa en el ordenador
  2. compilar el programa, y
  3. ejecutar el programa compilado.

Durante esta primera vez, cada una de esas etapas, necesitaras repetirlas unas cuantas veces para que le salga bien. No le daremos detalles de como realizar estas etapas, ya que dependen del ordenador en concreto y del entorno de desarrollo Java que este utilizando. Pero en general, puede escribir el programa utilizando cualquier editor de textos y salvándolo en un fichero. Entonces, debera usar algun comando para probar de compilar el fichero. Con ello, puede conseguir mensajes indicando que el programa contiene errores de sintaxis, o una version del programa compilado. En el caso de Java, el programa se compila en Java bytecode , no en código maquina. Finalmente, puede ejecutar el programa compilado por medio de comando apropiado. Para Java, actualmente deberá utilizar un interprete que ejecute Java bytecode. (Su entorno de programacion, puede que realize automáticamente, y sin su intervencion, alguno de estos pasos, pero puede estar seguro que estos tres pasos se han realizado.)

Aquí tenemos el programa Java que presentara el mensaje "Hola Mundo". No cuente todavia con entender que es lo que va hacer—algunas cosas, realmente no las entendera hasta dentro de algunos capítulos. 

Public class HelloWorld {
    // El programa presenta el mensaje
    // "Hola Mundo!" en la salida estandar
	public static void main (String[] args) {
       		System.out.println("Hola Mundo!");
	}
} //fin de la clase HelloWorld

La instrucción que realmente presenta el mensaje es:

System.out.println("Hola Mundo!");

Esta instrucción, es un ejemplo de una orden de llamada a subrutina. Utiliza una subrutina interna llamada System.out.println para realizar el trabajo. Recuerde que una subrutina consiste en un conjunto de instrucciones escritas para realizar una tarea, que hemos puesto juntas y les hemos dado un nombre. El nombre podrá ser usado para llamar (call) a la subrutina cada vez que necesitemos que se realice la tarea. Las subrutinas internas son las que ya vienen definidas como parte del lenguaje de programación, y por consiguiente, están disponibles automáticamente para usarlas en cualquier programa.

Cuando ejecute el programa , el mensaje "Hola Mundo!" (sin las comillas), se presentara en la salida estándar (standard output) . Por desgracia, no se exactamente en que forma! Java puede funcionar en muchas plataformas distintas y la salida estándar puede ser diferente en diferentes plataformas. Sin embargo, puede contar con que el mensaje se presente en algún sitio conveniente.

Puede que sienta curiosidad acerca de otras cosas del programa arriba indicado. Una parte del mismo, consiste en comentarios. Los comentarios en los programas, son totalmente ignorados por el ordenador, y están únicamente para ser leídos por los humanos. Esto no los hace menos importante. Los programas, han de poder ser leídos por las personas tan bien como por la máquina, y sin los comentarios, un programa seria muy difícil de entender. En Java, hay dos tipos de lineas de comentarios. El primer tipo, usado en el programa de arriba, empieza con //, y continua hasta el final de la linea. El ordenador ignora // y todo lo que le siga en la misma línea. El otro tipo de lineas de comentario, permiten continuar con el comentario durante mas de una línea. El comentario empieza con /* y termina con */.

Todo lo demas, lo requiere la sintaxis de Java. Cada programa debe definir una subrutina llamada main, esta definicion toma la forma :

public static void main(String[] args) {
instrucciones
}

La palabra public aquí, permite que esta subrutina pueda ser llamada desde fuera del programa. Esto es esencial, porque el interprete de Java, ejecuta el programa precisamente llamando a la subrutina main. El resto de esta primera línea es duro de explicar en este momento; por ahora, pensemos que es una parte requerida por la sintaxis. La definición de una subrutina – esto es, de las instrucciones que expresan lo que se quiere hacer—consiste en una secuencia de instrucciones (statements) encerradas entre dos llaves, { y }. Hemos usado instrucciones en el lugar en donde realmente se han de escribir las instrucciones. Siempre usaremos ese formato: Cualquier cosa que vea en negrita (y verde si su navegador soporta texto coloreado,) sera el lugar en donde debera escribir las instrucciones reales de su programa.

En Java, no existen las subrutinas de por si. Han de formar parte de una "clase"(class) . La subrutina main, debe formar parte de una clase publica con un aspecto semejante a este:

	public class nombre-programa {
		opcionalmente:variables,declaraciones y subrutinas
		public static void main(String[] args) {
			instrucciones
		}
            	opcionalmente:variables,declaraciones y subrutinas
        }

Dese cuenta que el nombre que aparece en la primera línea, es el nombre del programa, o bien, el nombre de la clase. Entonces, el nombre del programa "Hola Mundo!", es HelloWorld. Este programa, debe salvarse en un fichero llamado HelloWorld.java y cuando se compile, se generara el fichero HelloWorld.class. El fichero de clases resultante, contiene Java bytecode y puede ser ejecutado en el interprete de Java.

Tenga en cuenta, que el programa puede contener otras subrutinas ademas de la main(), y tambien unas cosas llamadas "declaracion de variables".Aprendera mas sobre ello mas adelante (empezamos con variables en la siguiente sección).
