Antes de empezar

Si no esta muy familiarizado con la interface de Windows 95, quizás sea el momento para que se tome unos minutos para conocerla. Debe entender como funciona la barra de tareas que aparece al pie de la pantalla y que puede tener un aspecto semejante a este:

En la esquina izquierda aparece el botón de Inicio (Start), que utilizamos para arrancar los programas, el resto de botones que aparecen, son las ventanas que tenemos activas en pantalla. Puede pasar de una ventana a otra pulsando con el ratos sobre la que le interese. (Si la ventana fuera visible, también podría pulsar con el ratón en ella, pero es mas normal utilizar la barra de tareas para cambiar de una a otra.)

En la mayoría de ventanas, podrá ver en su esquina superior derecha tres botones pulsando en el de la izquierda que aparece marcado como X, cerrara la ventana. El botón de la izquierda se utiliza para ocultar la ventana, Quedando entonces en la barra de tareas y si quiere volver a hacer visible la ventana, deberá pulsar en la barra de tareas. El botón del medio nos sirve para aumentar la ventana hasta ocupar toda la pantalla y para volverla a reducir dejándola en el tamaño que tenia antes de que ocupara toda la pantalla. El botón del medio cambia cuando la ventana ocupa toda la pantalla. Cuando este trabajando con los programas de este curso, va a tener que emplear un entorno de programación bastante complejo, que le obligara a ampliar la ventana para trabajar ocupando toda la pantalla, y aun necesitara espacio.

Para optimizar la cantidad de información que puede visualizar en su pantalla, puede comprobar cual es la resolución máxima que soporta su ordenador. La resolución de la pantalla se indica en el numero de pixeles de alto y de ancho. Cuantos mas pixeles, se podrán presentar mas cosa (el problema es que también se representan mas pequeñas. El limite se establece por lo que puede ver cómodamente) En un monitor de 14 pulgadas, no debe intentar visualizar mas de 800 x 600, ya que si no el tamaño de las cosas es tan pequeño que no será operativo, mientras que con un monitor de 17 pulgadas, puede llegar a 1024x768. Para cambiar la resolución de su monitor, debe seleccionar Inicio/Configuración/Panel de control, en esa pantalla deberá seleccionar el icono a continuación deberá señalar la pestaña de Configuración, y allí en el recuadro Area de escritorio, puede escoger la resolución a la que quiere trabajar, moviendo con su ratón, el cursor existente.

Si esta empezando con Windows 95, puede revisar el sistema de ayuda por medio del botón de Inicio/Ayuda.

Java y Visual J++

Los programas, se escriben en lenguajes de alto nivel como Java. A continuación, se deben traducir a lenguaje máquina antes de poderlos ejecutar. En el caso de Java, el programa se traduce a Java bytecode, que es una especie de lenguaje máquina pero independiente de la plataforma. Entonces el interprete ya puede ejecutar el programa en Java bytecode. (Para información acerca del lenguaje máquina, vea la Sección 1.1 de este curso, Para ver algo mas sobre el Java bytecode, vea la Sección 1.3)

La programación a menudo se realiza usando un Entorno de Desarrollo Integrado (Integrated Development Environment, o IDE). Un IDE es un programa que le permite realizar todos los pasos necesarios en la producción de un programa. Esto es , le permite escribir el programa, compilarlo, y ejecutar el programa resultante. En este curso, vamos a emplear el Visual J++ de Microsoft. Visual J++, se instala como parte de Microsoft Developper Studio que también soporta C++ y otros lenguajes. Esperamos conseguir que Microsoft, nos ceda una copia de libre distribución de la versión 1.0 de Visual J++, mientras tanto, puede ser interesante la copia de 30 días que ofrece Microsoft en http://msdn.microsoft.com/visualj/. (o una versión que se encuentra en el libro "Desarrollo de aplicaciones WEB al descubierto" de la Editorial Prentice Hall, ISBN:84-89660-80-8).

A partir de este momento, supondremos que ha podido conseguir una copia de Visual J++. Hay dos formas de arrancar el entorno de programación: Puede arrancarlo desde el botón Inicio/Programas/Microsoft Visual J++, o bien haciendo doble clic en cualquier fichero con extensión ".dsw" (Developper Studio Workspace), o en ficheros con extensión .java. Hemos mencionado el fichero de proyecto (.dsw)., este fichero lo crea el entorno de programación, para guardar información relevante del proyecto. Cuando quiere abrir un proyecto, haciendo doble-clic en ese fichero, se abrirá el J++, y presentara todos los ficheros del proyecto. Sin embargo, puede empezar a trabajar con un programa y el entorno le generara el fichero de proyecto automáticamente. Si tiene que empezar con un proyecto nuevo, ahora veremos como hacerlo.

A continuación,  puede bajarse el fichero practicas1 que contiene su primer proyecto en J++, seria buena idea que creara una carpeta en donde dejar todos estos programas que se va a bajar, o los que ira haciendo. Un poco de programación

En realidad, todavía no sabe suficiente  como para entender perfectamente el programa que hemos presentado en la primera parte es estas practicas. Pero es un programa lo suficientemente sencillo, como para que pueda imaginarse como funciona. Las líneas que realmente hacen "algo" -- preguntar el nombre e imprimir el saludo son:

           console.put("Por favor entre su nombre: ");
           name = console.getln();
           console.putln("Hola " + name + "! Encantado de conocerle.");

Estas líneas emplean una ventana de entrada salida  que llamo console. No es una parte estándar de Java, es algo que he escrito yo. Esta definido en el fichero Console.java que forma parte del proyecto de esta practica. Si el fichero no estuviera en el proyecto, el programa no podría realizar llamadas a la console. Puede leer mas acerca de todo esto en la Sección 2.4.

Los comandos console.put y console.putln, presentas string en la ventana de salida. La diferencia entre las dos, reside en que console.putln añade un carácter de retorno de carro al final de la string. El comando console.getln lee la línea que el usuario a estado tecleando. En este programa, la línea entrada, se asigna a la variable name con la instrucción de asignación "name=console.getln();". La variable es como una caja en donde guardaremos el valor. En la siguiente línea, cuando el ordenador encuentre la variable name, la sustituirá por el valor que hemos puesto dentro de la caja. Antes que una variable pueda ser usada, ha de ser declarada.. En este programa la variable name se declara con la instrucción "String name;".La palabra "String" en la declaración, significa que el valor que vamos a guardar en la caja es una cadena (string) de caracteres.

Con las asignaciones, pruebe de expandir el programa haciéndole otras preguntas al usuario e imprimiendo varias respuestas. Puede declarar otras variables para guardar cada una de las respuestas del usuario. Debe imitar las ordenes que hemos utilizado en el programa original. Tiene que escribir todo correctamente, incluidas las comillas, y terminando las líneas con el punto y coma (;). Una vez haya realizado esos cambios, ejecute nuevamente el programa. Cuando de la orden de Execute desde el menu Build el ordenador le avisara que tiene que volver a generar los ficheros. Esto se debe a que ha realizado cambios en el programa. Cuando haga clic sobre yes, el ordenador intentara compilar su programa. Si encuentra errores, los presentara en la parte inferior de la pantalla. Deberá corregir los errores y volver a probar. Como siempre, si tiene algún problema, pregunte.

Cuando consiga que su programa funcione, debe guardarlo con las modificaciones. Necesita el fichero modificado para enviarlo como parte del trabajo de practicas.

Los Webs

El World Wide Web, también llamado Web o WWW, consiste en millones de paginas de información pertenecientes a millones de ordenadores de todo el mundo, y que están entrelazadas por medio de los links. Ya se ha transformado en una fuente de información y de entretenimiento, y es algo que esta creciendo continuamente. Algo de la emoción de Java, es que fue creado como parte integrante de  los Web's. Suponemos que conoce y sabe usar correctamente los web's dado que esta leyendo estas notas que han sido pensada y escritas en forma de paginas Web.

Explorar los webs es algo muy entretenido. Una forma de realizarlo, es buscar una pagina de inicio, y luego ir siguiendo los enlaces (links) desde una pagina a otra. Un buen sitio para buscar las paginas de inicio, es Yahoo, con una cantidad enorme de paginas agrupada s por temas. Otra forma de buscar información, es utilizando palabras y unas herramientas que se llaman buscadores. Estas herramientas, permiten indicarle una o mas palabras, y rastrean las paginas web devolviendo una lista con las que contiene las palabras indicadas. Un ejemplo de esto es el buscador Alta Vista .(Después de seguir el link, marque (clic) el área de entrada, escriba las palabras que desee buscar y pulse el botón "Search" Se presentara una lista de enlaces para que pueda ir probándolos. Hay un sistema de ayuda en la misma pagina de Altavista, para que pueda aprender un poco mas sobre ella. Existen otros buscadores mas orientados por zonas geográficas; por ejemplo en España tenemos www.ole.es, www.ozu.es y otros muchos).

Las paginas se identifican en la red por medio de un URL o Uniform Resource Locator. En Netscape y en Internet Explorer la dirección de la pagina aparece en el recuadro "Dirección" en la parte superior de la ventana. Si conoce la dirección de la pagina que quiere, solo tiene que teclearla en ese recuadro y pulsar Enter. Por ejemplo, la pagina principal de este curso, esta disponible en la red en la dirección

http://members.xoom.com/elcurso

Esta pagina contiene los enlaces a todo el curso, y es a través de ella que vamos a poder ir comunicando las novedades que vayan apareciendo. Les aconsejamos que anoten esta pagina en Bookmarks (en Netscape) o en Favoritos (en Internet Explorer) para que les sea mas fácil volver en el futuro, y utilicen el servicio de marcas que facilita el curso, para poder continuar en el punto en que se encontraban.

Otro de los ejercicios de esta practica, es utilizar Altavista para buscar alguna pagina interesante (en castellano), y enviarnos la URL de esa pagina.

Applets y HTML

Java se diseñó para que pudiera integrarse perfectamente en Internet y en el World Wide Web. Una de las cosas que puede hacer con Java, es escribir applets, que son esos programas que pueden funcionar dentro de las paginas Web, en los navegadores Web como Netscape o Internet Explorer. (La programación de applets se empieza a tratar a partir del Capítulo 5 de este curso.) En este apartado de las practicas, nos limitaremos a realizar una primera exposición acerca de como trabajan los applets en Visual J++.

Necesitara una copia de la carpeta del proyecto Moire que se la puede baja e instala tal y como hemos comentado anteriormente; Una vez haya descomprimido el fichero bajado, se le habrá creado la carpeta prMoire, dentro se encuentra el fichero Moire1.dsw, haciendo doble-clic, se abrirá Visual J++. Vera que el proyecto tiene un único fichero, "Moire1.java". Este es el programa que funciona en una pagina del navegador (esto quiere decir que es un applet). Cuando de la orden de Execute desde el menu Build, vera que se le abre el navegado y se crea una pagina de forma totalmente dinámica, en la que aparece nuestro divertido applet.

Podríamos haber creado nosotros una pagina HTML que se llamara por ejemplo TestMoire, y haberla añadido al proyecto, con lo que tendría una forma semejante a esta:

Este panel sirve para organizar la información sobre su proyecto, y permitirle una forma sencilla de acceso. Puede mirarse un poco el programa Moire.java, para hacerse una idea del aspecto que puede tener un programa Java medianamente sofisticado, Sin embargo lo que nos interesa en este momento es el fichero HTML, TestMoire.HTML.

Un fichero HTML es algo semejante a un programa que se emplea para definir la pagina Web. Se escribe en un lenguaje que se llama HTML (HyperText Markup Language). En estos ficheros, se guarda todo el texto que aparece en una pagina web junto con etiquetas especiales (tags), o comandos para indicarle al navegador cómo tiene que aparecer el texto en pantalla, con que tamaño, y mas. También en este fichero pueden aparecer etiquetas que le indiquen al navegador (browser) que debe incluir otras cosas tales como imágenes o applets. Por ejemplo:

  • La etiqueta <hr> le indica al navegador que dibuje una línea horizontal ("horizontal rule")
  • La pareja de etiquetas <h2> y </h2> en <h2>Applet Moire</h2> le indican al navegador que las palabras "Applet Moire deben ser tratadas como titulares y deben aparecer en una línea de forma destacada. <h1> indicaría un titular mas importante, ósea mas destacado, y <h3> uno menos importante.
  • Las etiquetas <p> y </p> indican el principio y el final de un párrafo
  • La etiqueta <applet> y </applet> se emplean para incluir un applet en la pagina. La etiqueta applet incluye tres modificadores -- code, width y heigth-- que sirven para especificar el nombre del applet y el tamaño de la zona de la pagina que va a ocupar.

Pueden aparecer muchísimos mas tags. Considere también que la mayoría de personas que crean paginas web, no escriben ficheros HTML. En su lugar editan las paginas con alguna herramienta de creación de web como Netscape Composer, o Front Page. También hay muchos procesadores de texto, que son capaces de editar ficheros HTML. Sin embargo es necesario que conozcamos un poco sobre HTML para poder trabajar con applets.

Cuando ejecute el proyecto Moire, se abrirá el navegador con el fichero HTML. Pruebe de utilizar el comando Execute del menu Build o manteniendo pulsada la tecla "Control"(Ctrl) , pulse F5. Vera una pagina Web conteniendo el diseño animado llamado "Moire Pattern". Este diseño lo crea el applet. Cuando lo haya visto, cierre la ventana del navegador.

Una etiqueta <applet> puede contener parámetros que se utilizan para definir el applet. La idea general de utilizar parámetros para personalizar el applet es una de las cosas mas importantes que aprenderá con este curso.Los parámetros para un applet se especifican en la etiqueta <param>. Cada parámetro tiene un nombre y un valor. En el fichero HTML de ejemplo, hay dos parámetros llamados "lineColor" y "bgColor".  Cambiando el valor de estos parámetros, puede cambiar la apariencia o el comportamiento del applet. En el ejemplo, puede cambiar los valores de lineColor y bgColor para personalizar los colores del dibujo. Los valores validos para el color son: red, green, blue, cyan, magenta, yellow, black, white, gray, pink, y orange.

Puede probar diferentes valores para los parámetros de color. Además, el applet Moire tiene otros parámetros que pueden ser cambiados. En el fichero Moire.java, se encuentran documentados todos ellos, así como los valores por defecto que toman. Lea el código fuente, y pruebe algún otro parámetro al código HTML -- por ejemplo sleepTimer o border -- a la etiqueta applet en el fichero TestMoire.HTML. (Necesitara añadir una nueva etiqueta <param>).

Observe que la etiqueta del applet se refiere al fichero de clases (.class) que es el que contiene el código compilado y no al fichero que contienen el código fuente (.java)
