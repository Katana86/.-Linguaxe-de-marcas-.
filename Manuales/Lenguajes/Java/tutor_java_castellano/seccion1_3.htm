Sección 1.3: La maquina virtual Java

El código maquina, consiste en un conjunto de instrucciones muy simples que son ejecutadas directamente por la CPU del ordenador. Casi todos los programas son escritos en lenguajes de alto nivel como Java, Pascal, o C++. Un lenguaje de alto nivel, no puede ser ejecutado directamente por ningún ordenador. Primero es necesario trasladarlo a lenguaje maquina. Esta conversión, acostumbra a ser hecha por un programa llamado compilador. El compilador, toma las instrucciones en lenguaje de alto nivel, y las transforma en instrucciones ejecutables en código maquina. Una vez hecha esta conversión, el programa puede funcionar cualquier numero de veces, pero como es natural, solo en un tipo concreto de ordenador (cada tipo de ordenador, tiene su propio lenguaje maquina). Si el programa se ha de ejecutar en otro tipo de ordenador, deberá volver a ser convertido, usando un compilador diferente, al código maquina apropiado.

Existe una alternativa a este sistema de compilar el programa,. Se trata, de que en vez de utilizar un compilador que traduzca todo el programa de golpe, se utilice un interprete, que traduce instrucción por instrucción, a medida que va siendo necesario.Un interprete es un programa que trabaja de una forma muy semejante a la CPU, con una especie de ciclo de leer-y-ejecutar.Para poder ejecutar un programa, el interprete ejecuta un bucle en el que va leyendo las instrucciones una a una, decide si se han de ejecutar, y si es así, las convierte en el código maquina apropiado.

Un interprete se utiliza para ejecutar programas de alto nivel. Por ejemplo, el lenguaje de programación Lisp, en lugar de compilarse, es normalmente ejecutado por un interprete.Sin embargo, los interpretes pueden ser empleados de otra manera: permitir el uso de un programa en código maquina de un ordenador, en otro tipo de ordenador completamente diferente . Por ejemplo el programa llamado "Soft Windows" funciona en los ordenadores Macintosh. Soft Windows, es un interprete  que permite ejecutar programas escritos en código maquina de IBM-PC para ejecutarse bajo Windows 3.1.Si ejecuta Soft Windows en un Macintos, puede ejecutar cualquier programa Windows 3.1.(Desgraciadamente ira mucho mas lento que en un clonico IBM. El problema esta en que Soft Windows necesita emplear varias instrucciones maquina de Macintosh por cada instrucción PC que interpreta. Los programas compilados son normalmente mas rápidos que los interpretados).

Los diseñadores de Java, escogieron una combinación de interprete y compilación. Los programas escritos en Java se compilan a lenguaje maquina, pero es un lenguaje maquina para un ordenador que en realidad no existe.Este ordenador virtual es lo que se llama maquina virtual Java. El código maquina para la maquina virtual Java, es llamado Java bytecode. Para ejecutar los programas en Java bytecode, se utiliza un interprete. Naturalmente, son precisos distintos interpretes para cata tipo de ordenador,pero todos los programas Java  podrán funcionar en  cualquier ordenador que tenga un interprete para Java bytecode. Y el mismo Java bytecode podrá ser usado en cualquiera de estos ordenadores, independiente del tipo que sean, solo porque dispongan del interprete. Esta es la principal característica de Java:el mismo programa compilado, puede ser utilizado en distintos tipos de ordenadores.

Entonces, se preguntara asombrado, porque usar el producto intermedio , el Java bytecode?. Porqué no distribuir el programa original Java y dejar que cada persona se lo compile a lenguaje maquina o deje que su ordenador lo ejecute como quiera?.Hay mucha razones. La primera de todas, un compilador ha  de entender Java, un complejo lenguaje de alto nivel. El compilador en si mismo, es un programa complejo. Por otra parte, el interprete de Java, es un programa bastante pequeño y sencillo. Esto hace mas fácil el escribir un interprete de bytecode para cada nuevo tipo de ordenador, cuando sea necesario, permitiendo que esta nueva maquina pueda ejecutar cualquier programa Java, mientras que sería mucho mas duro escribir un nuevo compilador Java para esa misma máquina.

Además, los programas Java han sido pensados para transmitir por la red. Este problema ya concierne a la seguridad. Usted no querrá cargar un programa que pueda afectar a su ordenador o sus ficheros. El interprete de bytecode, actúa como una capa aislante entre usted y el programa descargado. Lo que se esta ejecutando realmente es el interprete, y el es el que hace funcionar de forma indirecta el programa descargado. El interprete le puede proteger de cualquier acción peligrosa que pueda intentar realizar el programa cargado.

Debemos destacar que la conexión entre Java y Java bytecode no tiene porque ser totalmente cerrada. Un programa escrito en Java, ciertamente puede ser compilado en código maquina de un ordenador real. Y programas escritos en otros lenguajes, podrían ser traducidos a Java bytecode. De todas maneras, es la combinación de Java y Java bytecode la que le permite programar  en un lenguaje de alto nivel orientado a objetos, independiente de la plataforma, seguro, y totalmente compatible con las redes.

 

Dos notas mas para concluir la sección:

  • Primero, Sun Microsystems -- el creador de Java -- esta trabajando en varios chips de ordenador que utilizarían Java bytecode como su código maquina real. Esto es , este chip seria una implementacion física de la maquina virtual Java. Esto haría que no fuera necesario el interprete de bytecode ya que seria interpretado a nivel hardware y la velocidad de ejecución mejoraría sustancialmente.
  • Segundo, debe tener en cuenta que la parte mas complicada de la independencia de plataforma, esta solucionada  por la interface gráfica "Graphical User Interface" -- con ventanas, botones,... --por lo que es necesario disponer de ella en cualquier plataforma que soporte Java.
